# Glossary

- `App layer`: the userspace Qt/QML application under `app/`
- `As-is architecture`: the current implemented architecture, documented without target-state overclaim
- `BSP`: board support package concerns such as overlay, kernel modules, and kernel integration assets
- `Brownfield`: work performed on an existing implementation rather than a blank-slate project
- `Current implementation`: behavior or structure that exists in source now
- `Future ideas`: broader project intent that is not yet committed as current or target scope
- `Product layer`: product-specific packaging and integration concerns intended for `yocto/meta-system-monitor/`
- `Target architecture`: the chosen intended design direction, not a statement that implementation is already complete
- `Transition architecture`: the migration state between current implementation and target direction
- `Yocto BSP layer`: `yocto/meta-rpi-ili9341-bsp/`, the active hardware-facing layer in this repository
