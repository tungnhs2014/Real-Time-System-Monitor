# Bring-Up Checklist

## BSP Validation
- overlay builds successfully
- display and touch modules are named consistently with recipes and source files
- target runtime loads the expected modules

## Hardware Validation
- display becomes active
- touch input registers correctly
- orientation and calibration are usable

## App Validation
- app starts
- dashboard renders on 320x240
- navigation is usable by touch
- settings persist

## Documentation Validation
- README points readers into docs
- docs match current paths and selected design direction
